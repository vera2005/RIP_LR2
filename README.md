Книжный каталог (Bookstore Lite) — REST API на Spring Bootэ

Этот проект представляет собой RESTful API для управления каталогом книг, разработанный в рамках лабораторной работы. Приложение построено на Spring Boot и использует Spring Data JPA/Hibernate для работы с данными. В качестве базы данных может использоваться как встроенная H2, так и PostgreSQL — выбор зависит от конфигурации. Проект реализует полный набор операций CRUD для книг и категорий, а также предоставляет дополнительные возможности: фильтрацию, поиск, пагинацию и работу с корзиной.

## Запуск приложения
1. Клонировать репозиторий
2. Перейти в папку проекта
3. Выполнить команду:
```bash
mvn clean spring-boot:run

Если вы планируете использовать встроенную базу данных H2, никаких дополнительных настроек не потребуется — база будет создана автоматически при запуске приложения. Все необходимые скрипты для инициализации данных находятся в файле src/main/resources/data.sql. 

 После успешного запуска приложение будет доступно по адресу http://localhost:8085

 Функционал приложения
API предоставляет полный набор операций для управления книгами и категориями. Для книг реализованы все базовые CRUD-операции: получение списка всех книг, получение информации о конкретной книге по её идентификатору, добавление новой книги, обновление существующей и удаление. Аналогичные операции доступны и для категорий.

Кроме того, реализованы функции фильтрации и поиска: можно получить список книг, принадлежащих определённой категории, а также выполнить поиск книг по автору или по названию. Для удобства работы с большими списками добавлена пагинация — при запросе списка книг можно указать номер страницы, количество элементов на странице и параметры сортировки.

Особенности реализации
Архитектура приложения следует многослойной структуре: контроллеры обрабатывают HTTP-запросы, сервисный слой содержит бизнес-логику, репозитории отвечают за взаимодействие с базой данных, а сущности отображают таблицы БД. Для передачи данных между слоями используются DTO (Data Transfer Objects), что позволяет отделить внутреннее представление данных от их внешнего представления в API.

Валидация данных осуществляется как на уровне сущностей с помощью аннотаций Bean Validation, так и на уровне бизнес-логики. Например, для поля isbn книги добавлена проверка на уникальность и соответствие формату. Все возникающие ошибки обрабатываются глобально с помощью @ControllerAdvice, что обеспечивает единый формат ответа об ошибках.

Для управления изменениями в структуре базы данных можно использовать Liquibase (опционально). Миграции хранятся в отдельной папке src/main/resources/db/migration/. Если используется H2, то база данных инициализируется с помощью SQL-скрипта data.sql.

Написаны unit-тесты для сервисов и интеграционные тесты для репозиториев и контроллеров, что позволяет убедиться в корректности работы всех компонентов. Документация API сгенерирована автоматически с помощью SpringDoc OpenAPI и доступна через Swagger UI.
